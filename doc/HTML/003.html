Telnet-Client für IConnect T2

3 Bedienung

Bevor T2 benutzt werden kann, muß mit IConnect bereits eine Verbindung zum Internet hergestellt worden sein. Dies geht derzeit noch nicht automatisch!

Wenn Sie nun auf TELNET.TTP doppelklicken, sollte Ihr Desktop einen Dialog anzeigen, in dem Sie eine Kommandozeile eingeben können. T2 erkennt verschiedene Kommandozeilenparameter, erwartet aber mindestens die Angabe eines Hosts.

T2 versucht nun, eine Verbindung zum gewünschten Host aufzubauen. Falls dies aus irgend einem Grund nicht möglich ist, wird eine der Fehlermeldungen ausgegeben. Ansonsten können sie nun auf dem entfernten Rechner in einem Atari-Fenster arbeiten! Wie dies geschieht, sollte jeder, der Telnet einsetzt, bereits wissen.

Beendet wird T2, indem entweder mit IConnect die Verbindung zum Internet beendet wird oder man sich aus dem Host ausloggt (meistens mit dem Kommando "exit") - T2 erkennt dies und beendet sich dann automatisch. Bei letzterer Variante muß man die Telefonverbindung dann aber immer noch manuell mit IConnect trennen!

3.1 Kommandozeilenparameter

Folgende Parameter sind in der Kommandozeile erlaubt:

  • wenn der Parameter "-d" (natürlich ohne Anführungszeichen) in der Kommandozeile vorhanden ist, wird im Verzeichnis von TELNET.TTP die Datei "telnet.log" angelegt, in der alle empfangenen Zeichen mitprotokolliert werden - insbesondere sollten dort also keine Paßwörter auftauchen.
     
  • alle anderen Parameter werden als Hostangaben aufgefaßt, wobei jedoch nur zum zuletzt angegebenen Host eine Verbindung aufgebaut wird. Ein Host bezeichnet einen Rechner im Internet, der entweder durch seinen Klartextnamen (z.B. "rzstud1.rz.uni-karlsruhe.de") oder durch seine IP-Adresse in der Form "a.b.c.d" (z.B. "129.13.95.1") angegeben wird.
     

Gültige Kommandozeilen wären also (jeweils wieder ohne Anführungszeichen)

  • "meinrechner.meinedomain.de"
     
    Es wird eine Verbindung zum Host "meinrechner" hergestellt.
     
  • "-d meinrechner.meinedomain.de"
     
    Es wird eine Verbindung zum Host "meinrechner" hergestellt und die Protokolldatei "telnet.log" angelegt.
     
  • "meinrechner.meinedomain.de hostxyz.org -d"
     
    Es wird eine Verbindung zum Host "hostxyz" hergestellt und die Protokolldatei "telnet.log" angelegt.

3.2 Internes

  • Da T2 im Moment das MagiC beiliegende VT52.PRG (oder eine passende andere VT52-Emulation) voraussetzt, wird dem Server auch nur "VT52" als Terminalemulation gemeldet (aber immerhin kein Zeilendruckermodus oder sonstwas prähistorisches).
     
  • Die Größe des Terminals wird von VT52.PRG übernommen (genauergesagt aus den Environmentvariablen LINES bzw. ROWS und COLUMNS; wenn diese Variablen nicht existieren, werden die Defaultwerte 24 und 80 verwendet). Hier gibt es derzeit noch eine kleine Einschränkung: Die maximale Anzahl der Zeilen bzw. Spalten kann 254 noch nicht überschreiten (und wird daher auf diesen Wert begrenzt).
